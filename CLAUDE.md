## Project Overview

Jekyll blog with a fully custom cyberpunk dark theme (originally forked from jekyll-theme-chirpy, now entirely bespoke). Deployed to GitHub Pages at 0xdeadbeefjerky.com. Cybersecurity/offensive security blog by Dominic Spinosa.

## Commands

Ruby is not installed on the host machine. Use Docker for all build/serve/test operations:

```bash
# Start a new development container (installs deps, builds, and serves on port 4001)
docker run -d --name jekyll-preview -v $(pwd):/srv/jekyll -p 4001:4000 ruby:3.1-slim \
  bash -c "apt-get update && apt-get install -y build-essential git && cd /srv/jekyll && bundle install && bundle exec jekyll serve --host 0.0.0.0 --port 4000"

# Build inside a running container
docker exec jekyll-preview bash -c "cd /srv/jekyll && bundle exec jekyll build -d _site"

# Test (HTML validation — also runs in CI)
docker exec jekyll-preview bash -c "cd /srv/jekyll && bundle exec htmlproofer _site --disable-external --check-html --allow_hash_href"
```

Note: The container git ownership warning (`fatal: detected dubious ownership`) is harmless — it only affects the `posts-lastmod-hook.rb` plugin's ability to read git history inside the container.

After modifying `_config.yml`, the serve process must be restarted (config changes are not picked up by `--watch`).

## Architecture

### Layout Hierarchy

All layouts extend `default.html` which provides the HTML shell, nav, footer, search overlay, and JS loading.

- `home.html` — paginated post card grid with hero section
- `post.html` — single post with breadcrumbs, series nav, reading progress bar, related posts, share buttons, prev/next nav, and sticky ToC sidebar
- `page.html` — simple single-column content (used by `_tabs/` pages)
- `blog.html` — year-grouped archive list
- `categories.html` / `tags.html` — taxonomy index pages
- `category.html` / `tag.html` — individual taxonomy pages (generated by `jekyll-archives`)

### Key Includes

- `head.html` — meta tags, `{% seo %}`, structured data, Font Awesome CDN, Google Analytics (production only)
- `nav.html` — fixed top nav with avatar, tab links, search toggle button, mobile hamburger
- `search.html` — full-screen search modal overlay (Simple-Jekyll-Search)
- `related-posts.html` — pure Liquid scoring by shared tags (weight 2) and categories (weight 1)
- `series-nav.html` — multi-part post series navigation (activated by `series:` front matter)
- `breadcrumbs.html` — Home / Category / Post Title breadcrumb trail
- `structured-data.html` — enriched Article JSON-LD beyond jekyll-seo-tag
- `toc.html` — sticky sidebar table of contents (JS-generated from h2/h3 headings)

### Custom Plugins

- `_plugins/posts-lastmod-hook.rb` — auto-sets `last_modified_at` from git history
- `_plugins/lazy-images.rb` — post-render hook injecting `loading="lazy"` on all `<img>` tags

### Styling

Fully custom SCSS (not using upstream Chirpy CSS). Dark-only theme with neon cyberpunk accents. All design tokens are CSS custom properties in `_sass/_variables.scss`:

- Colors: `--accent-cyan` (#00fff0), `--accent-magenta` (#ff00aa), `--accent-green` (#39ff14), etc.
- Fonts: Orbitron (display), Space Grotesk (headings), Inter (body), JetBrains Mono (code) — loaded via Google Fonts CDN
- Spacing scale: `--space-xs` through `--space-3xl`

SCSS partials are imported in `assets/css/main.scss`. New partials go in `_sass/` and must be added to the import list.

### JavaScript

Single vanilla JS file at `assets/js/main.js` (no build system). All features live inside one `DOMContentLoaded` handler. Features: nav toggle, tagline typewriter, code copy buttons, ToC generation + scroll spy, reading progress bar, search modal (Ctrl+K shortcut), scroll fade-in animations.

### Search

Client-side search using Simple-Jekyll-Search. `search.json` (repo root) is a Liquid template that generates the post index at build time. The JS library is loaded from unpkg CDN in `default.html`.

## Post Front Matter

```yaml
---
title: "Post Title"
description: >
    Brief description for SEO/previews.
date: YYYY-MM-DD HH:MM:SS-05:00
categories: [Category1, Category2]
tags: [tag1, tag2, tag3]
toc: true
image: /assets/img/post-image.webp
series:                              # optional, for multi-part posts
  name: "Series Name"
  part: 1
---
```

- `image:` is used by jekyll-seo-tag for Open Graph/Twitter cards. Falls back to `/assets/img/og-default.png` (set in `_config.yml` defaults).
- `series:` activates the series navigation component. Posts are grouped by `series.name` and ordered by date.
- Categories and tags use bracketed arrays. TOC defaults to true.
- Drafts go in `_drafts/`.

## Conventions

- Dark-only theme (no light mode toggle).
- `.editorconfig`: 2-space indentation (4-space for JS), UTF-8, LF line endings.
- CI runs on Ruby 3.1 via GitHub Actions: builds site, runs htmlproofer, deploys to Pages.
- Custom domain configured in `CNAME` file.
- Data-driven social sharing (`_data/share.yml`) and footer contacts (`_data/contact.yml`).
